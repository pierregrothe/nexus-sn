# Story 2026.05.01: CLI adaptive list rendering + terminal capability detection

Status: ready
Spec-Clarity: high
Depends-On: none

## Story

As a NEXUS operator running any command on any terminal,
I want NEXUS to detect my terminal's capabilities at startup and
pick the best rendering strategy automatically -- no flags, no
configuration -- so that `nexus plugins list` scrolls with sticky
header and `/search` on my modern terminal, prints ASCII boxes on
my legacy cmd.exe, and dumps tab-separated rows when I pipe to
`grep`.

## Acceptance Criteria

AC1:
**Given** a modern TTY (Windows Terminal / iTerm / gnome-terminal
/ kitty / VS Code) and an inventory with rows that would overflow
the visible area
**When** the user runs `nexus plugins list`
**Then** the table is rendered through `pypager` with sticky
header, `/search`, `q` to quit, arrow / PageUp / PageDown
navigation. The selected profile is RICH.

AC2:
**Given** a modern TTY and an inventory that fits the visible
area (`len(rows) <= caps.rows - 4`)
**When** the user runs `nexus plugins list`
**Then** the table is rendered inline (no pager). Selected
profile is RICH.

AC3:
**Given** a TTY with 16-color or 256-color support (no truecolor)
or terminal height < 24
**When** the user runs `nexus plugins list`
**Then** the table renders without gradient borders, using
16-color theme tokens; pager invoked when overflow. Profile is
BASIC.

AC4:
**Given** a pre-Windows-Terminal legacy cmd.exe (`legacy_windows
== True`)
**When** the user runs `nexus plugins list`
**Then** the table renders as ASCII box-drawing, inline (no
pager invocation, since pypager requires VT escapes). Profile is
LEGACY.

AC5:
**Given** stdout redirected to a file / pipe, or `$TERM=dumb`,
or `$CI / $GITHUB_ACTIONS / $JENKINS_HOME / $GITLAB_CI / 
$BUILDKITE / $CIRCLECI / $TRAVIS / $DRONE / $TF_BUILD` is set
**When** the user runs `nexus plugins list`
**Then** the table renders as tab-separated rows, one per line,
no ANSI escapes, no pager. Profile is PLAIN.

AC6:
**Given** any TTY
**When** the user runs `nexus plugins list --plain` OR
`NEXUS_PLAIN=1 nexus plugins list`
**Then** PLAIN profile is forced regardless of detection.

AC7:
**Given** any TTY with `$NO_COLOR` set
**When** the user runs `nexus plugins list`
**Then** color is dropped (BASIC or LEGACY profile depending on
other factors), pager still invoked on overflow when not LEGACY.

AC8:
**Given** Windows 11 with Windows Terminal AND a legacy app
context that confuses Rich's `legacy_windows` detection
**When** `$WT_SESSION` is set
**Then** `legacy_windows` is forced False; profile picks
RICH/BASIC correctly.

AC9:
**Given** stdout redirected (`os.get_terminal_size()` raises
`OSError`)
**When** `TerminalCapabilities` is built
**Then** `cols=80, rows=24` are used as fallback values; no
exception leaks.

AC10:
**Given** `_argv_has_plain()` is called with `sys.argv =
["nexus", "plugins", "list", "--plain"]`
**When** `make_console()` runs at process startup (before
Typer parses argv)
**Then** the function returns True; `caps.forced_plain` is True;
profile is PLAIN.

AC11:
**Given** any `nexus` command
**When** the user runs `nexus status`
**Then** a "Terminal" panel renders showing
`Profile: RICH | TTY: yes | CI: no | Color: TRUECOLOR | Size:
120x40 | Terminal: WindowsTerminal | Pager: pypager`.

AC12:
**Given** the test suite
**When** unit tests for `pick_profile` run
**Then** they test all four profile outcomes by constructing
`TerminalCapabilities` instances directly (no env-var manipulation,
no `Console` instantiation).

AC13:
**Given** the test suite
**When** unit tests for `PagedTable.render` run
**Then** they use `FakePager` (records the renderable handed to
`page()`) AND `RenderContext` instances with hand-built
`TerminalCapabilities` for each of the four profiles.

AC14:
**Given** any TTY in tmux or screen multiplexer
**When** `$COLORTERM=truecolor` is NOT set
**Then** profile downgrades RICH -> BASIC even if color depth
claims truecolor.

## Must NOT

- Must NOT add Textual or any TUI framework as a dependency.
- Must NOT shell out to system `less` / `more` / `pydoc.pager`.
  Pager backend is `pypager`, period.
- Must NOT add `--pager / --no-pager / --limit / --no-color` CLI
  flags. The ONLY new flag in this story is `--plain`.
- Must NOT attach attributes (`_nexus_caps`, `_nexus_profile`) to
  the Rich `Console` instance. RenderContext is threaded through
  Typer's `ctx.obj`, retrieved via `get_render_context(ctx)`.
- Must NOT modify the existing `DataTable` public surface
  (`DataColumn`, `DataTable.add_row`, `from_rows`). `PagedTable`
  composes DataTable; does not replace it.
- Must NOT cache profile detection across NEXUS invocations
  (~1 ms cost; re-runs keep the model honest).
- Must NOT migrate existing non-target commands (Status,
  Capture, Instance) to RenderContext in this story. Only
  `nexus plugins list` and `nexus status` are touched.
- Must NOT use `# type: ignore` to bypass pyright strict on
  Console attribute access. Use `RenderContext` instead.

### Testing Approach

- Real fakes only. `FakePager` records calls. `Console(
  force_terminal=True/False, color_system="truecolor", file=
  StringIO())` for TTY-routing tests.
- `pick_profile` is pure: feed hand-built `TerminalCapabilities`,
  assert the four profiles -- no env-var manipulation needed in
  these tests.
- `detect()` tests use `monkeypatch.setenv` + Rich Console with
  `file=StringIO()` to exercise the env-var matrix.
- `_argv_has_plain()` is pure: feed argv list directly, assert
  bool.
- Coverage gate: 100% line for all new files. ratchet.json
  baselines added.

### Conventions

- File headers on every new `.py` file (`# src/nexus/...`).
- Pydantic models frozen + strict + extra="forbid".
- Google-style docstrings on every public function / class.
- `__all__` on every module.
- Absolute imports only.
- UTC for any timestamp (none in this story but rule stands).
- No `# type: ignore` (the whole point of this refactor).
- ASCII-only.
- Test naming: `test_<function>_<scenario>`.
